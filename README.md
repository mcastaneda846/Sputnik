# Challenge HTML - Ejercicio Colaborativo de Git

## Descripción del Proyecto

Este repositorio es un ejercicio práctico diseñado para aprender el flujo de trabajo colaborativo con Git y GitHub. El objetivo es que **4 miembros de un equipo** trabajen juntos para construir una página HTML básica, donde cada persona contribuye con una parte específica del código.

## Objetivos de Aprendizaje

- Crear y gestionar ramas (branches)
- Realizar commits significativos
- Crear y revisar Pull Requests (PR)
- Hacer merge de código
- Resolver conflictos de merge
- Trabajar en equipo usando Git/GitHub

## Estructura del Equipo

El equipo está dividido en 4 roles:

### **Miembro 1: Estructura Base**
- Crear la estructura HTML básica
- DOCTYPE, etiquetas `<html>`, `<head>`
- Meta tags y título del documento

### **Miembro 2: Header y Navegación**
- Crear el `<body>` y estructura principal
- Implementar header con título principal
- Agregar barra de navegación

### **Miembro 3: Contenido Principal**
- Crear sección de contenido
- Agregar párrafos descriptivos
- Insertar imágenes

### **Miembro 4: Footer y Estilos**
- Implementar footer
- Agregar información de contacto
- Crear estilos CSS básicos

## Estructura del Repositorio

```
challenge-html/
├── README.md                    # Este archivo
├── CHALLENGE.md                 # Guía detallada del ejercicio
├── index.html                   # Archivo HTML principal (se construye colaborativamente)
├── docs/
│   ├── GUIA_MIEMBRO_1.md       # Guía para Miembro 1
│   ├── GUIA_MIEMBRO_2.md       # Guía para Miembro 2
│   ├── GUIA_MIEMBRO_3.md       # Guía para Miembro 3
│   ├── GUIA_MIEMBRO_4.md       # Guía para Miembro 4
│   └── RESOLUCION_CONFLICTOS.md # Guía para resolver conflictos
└── assets/
    └── styles.css               # Archivo CSS (creado por Miembro 4)
```

## Inicio Rápido

1. **Clona el repositorio**
   ```bash
   git clone <URL_DEL_REPOSITORIO>
   cd challenge-html
   ```

2. **Lee la guía completa**
   - Revisa `CHALLENGE.md` para instrucciones detalladas
   - Identifica tu rol en el equipo (Miembro 1, 2, 3 o 4)
   - Lee tu guía específica en `docs/GUIA_MIEMBRO_X.md`

3. **Comienza a trabajar**
   - Crea tu rama
   - Realiza tus cambios
   - Crea un Pull Request
